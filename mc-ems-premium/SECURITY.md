# Security Policy

## Reporting a Vulnerability

We take the security of MC-EMS seriously. If you discover a security vulnerability, please follow our responsible disclosure policy.

### How to Report

**Please DO NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email your security concern to:

📧 **Email**: security@example.com

**Include in your report:**
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if applicable)

### Response Timeline

- **24 hours**: Initial acknowledgment
- **72 hours**: Security assessment and impact evaluation
- **7 days**: Security fix development and testing
- **14 days**: Coordinated disclosure and patch release

### Supported Versions

Security updates will be provided for:

| Version | Supported | End of Life |
|---------|-----------|------------|
| 1.0.x   | ✅ Yes    | 2027-03-13 |

### Security Best Practices

When using MC-EMS:

1. **Keep WordPress Updated** - Always run the latest WordPress version
2. **Keep PHP Updated** - Use PHP 7.4 or higher
3. **Use Strong Passwords** - Enforce strong passwords
4. **Regular Backups** - Maintain regular backup schedules
5. **Limited User Roles** - Grant minimum required permissions
6. **Update Plugins** - Keep all plugins updated

### Disclosure Policy

Once a security vulnerability is fixed:

1. A patch version is released
2. Security advisory is published
3. Affected users are notified
4. Public disclosure occurs with proper credit
